01 — Concept

L'entreprise qui se pilote seule

0Dépense repose sur un principe simple : chaque IA a un rôle précis, une mission définie, et communique avec les autres selon un protocole strict. Le résultat : une machine à revenus entièrement automatisée.

🔬

Recherche

ZARA identifie en continu des opportunités de revenus réels avec budget zéro. Freelance, affiliation, contenu, automatisation — toutes les pistes sont explorées.

🛡️

Contrôle

Chaque idée et chaque production passe par un contrôleur dédié. Aucun projet ne démarre sans validation. Aucun contenu ne sort sans audit.

⚡

Exécution

Une fois validé, le projet est exécuté par les IA spécialisées — programmation, design, rédaction — de façon coordonnée et tracée.

🗂️

Mémoire

MEMO archive chaque décision, chaque rapport, chaque résultat. Le projet 0Dépense ne perd jamais d'information.

📡

Communication

IRIS assure que chaque IA sait ce qu'elle doit faire, quand le faire, et à qui transmettre son travail. Aucune tâche ne reste sans destinataire.

📈

Croissance

Les revenus générés financent de nouveaux projets. Le système se nourrit de lui-même et monte en puissance sans jamais nécessiter de capital extérieur.

03 — Flux de travail

De l'idée au revenu

Chaque projet suit le même protocole strict, de la recherche à l'exécution. Aucune étape ne peut être sautée.

01

ZARA trouve une idée

La chercheuse identifie une opportunité de revenu à budget zéro et produit une fiche complète : faisabilité, potentiel, première action, outils gratuits nécessaires.

02

VERA audite le projet

La contrôleure N°1 vérifie les 3 règles fondamentales : budget 0€, conformité légale, potentiel de revenus réel. Si une règle est violée → retour à ZARA. Si tout est bon → validation.

03

MEMO archive la décision

L'archiveur crée une fiche numérotée (ex: 0D-007) avec le statut du projet, le résumé du rapport VERA et l'historique complet. Le projet est tracé à vie.

04

IRIS distribue les tâches

La secrétaire reçoit le projet validé et envoie les instructions aux IA concernées : ATLAS pour le technique, PIXIE pour le visuel, PEN pour le contenu écrit.

05

Équipe créative produit

PIXIE crée les visuels (→ validés par NOVA), PEN rédige les contenus (→ validés par REX), ATLAS construit les outils techniques. Chaque production est contrôlée avant livraison.

06

0Dépense reçoit le rapport final

Le centre de contrôle reçoit le projet complet, validé à chaque étape. Il décide du lancement. MEMO met à jour l'archive. Les revenus générés alimentent les prochains projets.

04 — Règles fondamentales

3 règles.
Aucune exception.

Ces 3 règles s'appliquent à chaque projet, sans négociation possible. Elles sont vérifiées par VERA à chaque soumission.

01

Budget Zéro

Aucun projet ne peut nécessiter le moindre euro d'investissement de départ. Seuls les outils gratuits sont autorisés. Le moindre coût entraîne un refus immédiat.

02

Légalité Totale

Chaque projet doit respecter toutes les lois en vigueur : droit du travail, fiscalité, propriété intellectuelle, RGPD. Aucun risque légal n'est acceptable.

03

Revenus Réels

Le projet doit avoir un potentiel de revenus significatif et vérifiable. Les idées à gain marginal ou incertain sont refusées. Seul ce qui rapporte vraiment est lancé.

Python — Communication ZARA → VERA via API 
# ATLAS — Pont de communication automatique entre les IA
# Envoie le rapport de ZARA à VERA pour validation

import requests
import json

# Config API Gemini (gratuite)
GEMINI_URL = "https://generativelanguage.googleapis.com/v1beta/models/gemini-pro:generateContent"
API_KEY    = "VOTRE_CLE_GRATUITE"

def zara_cherche_idee():
    prompt = """
    Tu es ZARA du projet 0Dépense. Trouve 1 idée de revenu à budget 0€.
    Format: NOM | CATÉGORIE | ACTION_24H | REVENU_ESTIMÉ | AUTOMATISABLE
    """
    response = appel_gemini(prompt)
    return response

def vera_valide(idee):
    prompt = f"""
    Tu es VERA du projet 0Dépense. Audite cette idée en 3 règles:
    IDÉE: {idee}
    Règle 1 - Budget 0€? Règle 2 - Légal? Règle 3 - Revenu réel?
    Verdict: VALIDÉ ou REFUSÉ + raison
    """
    response = appel_gemini(prompt)
    return response

def appel_gemini(prompt):
    payload = {"contents": [{"parts": [{"text": prompt}]}]}
    r = requests.post(GEMINI_URL, params={"key": API_KEY}, json=payload)
    return r.json()["candidates"][0]["content"]["parts"][0]["text"]

# Pipeline automatique
if __name__ == "__main__":
    print("[ATLAS] Démarrage du pipeline 0Dépense...")
    idee    = zara_cherche_idee()
    print(f"[ZARA] {idee}")
    verdict = vera_valide(idee)
    print(f"[VERA] {verdict}")
Python — MEMO : Archivage automatique des projets 
# ATLAS — Système d'archivage automatique pour MEMO
# Sauvegarde chaque projet dans Supabase (gratuit)

import json, datetime
from supabase import create_client

supabase = create_client("VOTRE_URL_SUPABASE", "VOTRE_CLE_GRATUITE")
compteur = 1  # Numéro auto-incrémenté

def memo_archive(ia_source, nom_projet, statut, resume):
    global compteur
    fiche = {
        "dossier"  : f"0D-{str(compteur).zfill(3)}",
        "date"     : datetime.date.today().isoformat(),
        "source"   : ia_source,
        "projet"   : nom_projet,
        "statut"   : statut,   # validé / refusé / en_cours
        "resume"   : resume,
        "archive"  : "MEMO ✓"
    }
    supabase.table("projets_0depense").insert(fiche).execute()
    print(f"[MEMO] Dossier {fiche['dossier']} archivé — {nom_projet}")
    compteur += 1
    return fiche["dossier"]

def memo_etat_lieux():
    data = supabase.table("projets_0depense").select("*").execute()
    projets = data.data
    print(f"[MEMO] Total archivés: {len(projets)}")
    for p in projets:
        print(f"  [{p['dossier']}] {p['projet']} — {p['statut']}")

# Exemple d'utilisation
memo_archive("VERA", "Bot Telegram Affiliation", "validé", "Bot automatique, 0€, légal, revenu estimé 200€/mois")
